# MicroAgent 项目定位与核心目标

> 最后更新：2026-04-10

---

## 项目定位

MicroAgent 是一个**面向普通 Windows 用户的本地 AI Agent**，核心命题是：

> 把 AI Agent 的能力装进一个 .exe 文件，放在桌面上双击就能用。

目标用户两类：
- **开发者自用**：处理文件、查资料、系统监控等日常任务，需要可定制、可扩展
- **分发给他人**：打包后分享给非技术用户，要求开箱即用，零安装门槛

---

## 硬件约束（设计出发点）

**基准机器：AMD R7 8845H + 16GB RAM + Windows 11**

- iGPU：Radeon 780M，支持 Vulkan，显存共享系统内存
- 没有独立显卡，无法运行主流 70B/30B 大模型
- 内存有限，模型加载后需控制在 4GB 以内
- 轻薄本，需控制 CPU 线程数，不能霸占系统资源

这个约束不是限制，是设计驱动力：**所有技术选型都为这台机器服务**。

---

## 设计原则

**1. 轻量第一**
零额外运行时依赖是硬约束。打包后 exe 内置 Python，用户只需放好模型文件即可运行。每引入一个第三方依赖都要问：能否用标准库替代？收益是否值得体积代价？

**2. 易用性优于简洁性**
代码可以复杂，用户体验不能复杂。愿意用更多代码换取更低 token 消耗、更好的模型表现、更少的用户操作。

**3. 功能完整，实现可简单**
接口设计要完整（stub 也算），不做残缺的半成品。今天的简单实现不妨碍明天的升级——接口稳定，替换实现即可。

**4. 扩展而非配置**
新工具 = 继承基类 + 注册。新后端 = 实现 ABC。不为假设中的需求做配置开关，为真实的扩展场景留好接口。

## 关键技术选型及理由

| 组件 | 选型 | 理由 |
|---|---|---|
| 推理后端 | llama-cpp-python（GGUF） | 支持 Vulkan/CUDA，可源码编译，Nuitka 友好 |
| 主力模型 | Gemma-4 E2B Instruct | 2B 级移动端优化，128K context，工具调用能力强 |
| Agent 框架 | smolagents ToolCallingAgent | 极致轻量，LLM 输出结构化 JSON 比生成 Python 代码对小模型更可靠 |
| CLI 渲染 | rich | 成熟、稳定，彩色输出零配置 |
| 配置校验 | pydantic | 错误提示友好，schema 即文档 |
| 打包 | Nuitka onefile | 性能优于 PyInstaller，比反编译更难，内置 Python 运行时 |
| 内存数据库 | SQLite（stdlib） | 零依赖，WAL 并发，事务原子，未来可换后端 |

**刻意不选的：**
- ChromaDB / ONNX 向量库：与 Nuitka 不兼容
- 云端 API 模型：违背本地化原则
- Electron / Web UI（当前阶段）：体积代价过大，CLI 够用

## 分发形态

三个构建变体，覆盖不同 GPU 环境：

| 变体 | 适用硬件 | 加速方式 |
|---|---|---|
| `cuda` | NVIDIA GPU | CUDA，需源码编译（支持 Gemma-4 架构） |
| `vulkan` | AMD Radeon / Intel iGPU | Vulkan，轻量，开箱即用 |
| `cpu` | 无 GPU | 纯 CPU，降级运行 |

用户拿到的是：一个 exe + 一个 config.yaml + 一个 models/ 目录。

---

## 功能边界

**当前范围（CLI MVP）：**
- 本地模型加载与推理
- 工具调用：网页搜索、文件操作、系统信息
- 跨 session 记忆（SQLite 持久化）
- 会话上下文压缩（阈值触发 + 手动命令）
- 彩色 CLI 交互

**明确不在当前范围：**
- Web UI / 桌面 GUI（接口已预留，未来可接入）
- 多 Agent 协作（架构已预留，未来扩展）
- 云端部署
- 移动端

---

## 演化方向

阶段一（当前）→ 阶段二（Web/GUI）→ 阶段三（多 Agent）

架构设计保证阶段间平滑升级：`AgentRunner` 是接口，CLI 是它的一个消费者，Web UI 可以平行接入同一接口而不改动核心逻辑。
